Secure Socket Layer

Written By: Jaeden Fuller | October 17th, 2021

Hassan, M (Unstated) "Cyber Security" [CC0] Public Domain Pictures

Have you ever opened up a website and noticed that lock icon that appears in the address bar, right before the URL, but never really knew what it did? If so, you’ve come to the right post!

Secure Socket Layer?

Secure Socket Layer (SSL) is a network security protocol that was first developed in 1995 by a company called Netscape. The purpose of SSL was to ensure safe communication over the internet via encryption & authentication. Although, SSL had many security flaws. That’s why in 1999 SSL was replaced by Transport Layer Security (TLS), the name change happened because the Internet Engineering Task Force (An international community of web experts with the goal of furthering the operation of the Internet) was taking over development of the protocol. Currently, TLS is the network security protocol used by most people, with the newest version, TLS 1.3, being released in August 2018.

Am I Using It?

Yes! You might not know it, but every time you open a webpage and see that lock icon in the address bar that means you are using Hypertext Transfer Protocol Secure (HTTPS) which in fact uses TLS/SSL to authenticate the website and to encrypt all the data you send. If you don't see that lock icon in the address bar it means you are only using Hypertext Transfer Protocol (HTTP).

Lock Icon

HTTP isn't necessarily dangerous, you can still watch videos or read articles on websites that use it, but you don't want to input any sensitive information in websites that only use it. The reason being that HTTP has no encryption and your information would be sent to the server in plain text and extremely easy for anyone to steal.

If you want to learn more about internet protocols or cybersecurity in general you could take this Cisco course on the topic, or you could take one of the many cyber-focused courses on cybrary.

Glossary:

Network Security Protocol: Network Security Protocols are the protocols used to protect your data while you are sending it across a network.

Encryption: “The process of converting information or data into a code, especially to prevent unauthorized access” - Oxford dictionary definition.

Ex. I want to pass a note to you that says “Hello” but I don't want anyone else to be able to read it. I would encrypt it by shifting all the letters from their placement in the alphabet by one to the right. H -> I, E -> F, L -> M, L -> M, O -> P. My end result would be IFMMP.

Authentication: Authentication is the act of verifying the user/server.

Plain Text: Plain text refers to un-encrypted text/text that would make sense to the person reading it.

Ex. This is written in plain text, XLMWMWRSXTPEMRXIBX (Decrypted: This is not plain text).
